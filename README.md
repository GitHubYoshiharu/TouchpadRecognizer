# TouchpadRecognizer
タッチパッドの2本指タップを検知し、その指の位置に対応するキーを入力する、デスクトップアプリケーションです。
入力されたキーをAutoHotKeyのホットキーに設定することで、タッチパッドをランチャーとして使えるようにすることを想定しています。

# 簡単な仕組み
1. タッチパッドに指が触れると、タッチパッドから指の座標・座標の最大値を含むデータが送られてくる
2. 座標の最大値を用いてタッチパッドを5つの判定エリアに分割する
3. 一度触れた指が一定時間内に離れたら、タップ操作が行われたと判定する
4. 触れていた指がそれぞれどのエリアに属していたかを、指の座標を基に判定する
5. 指の属していたエリアの組み合わせに応じて、対応するキーを入力する

# 使い方
このアプリはAutoHotKeyとの併用を想定して作られていますが、連携する機能は無いので、併用する際はAutoHotKeyのスクリプトファイル（ahkファイル）を用意し、個々に起動させる必要があります。
このアプリは、2本の指が触れていたエリアの組み合わせによって、11種類のキー（F13～F23）を入力します。
F13～F23キーをAutoHotKeyでホットキーに設定することで、タップするエリアによってAutoHotKeyの処理を呼び分けられます。

このアプリケーションは起動すると、ウィンドウなどを表示せず、タスクトレイに残ります。
タスクトレイアイコンを左クリックするとメニューが表示され、「終了」ボタンから明示的にアプリを終了できます。
メニューには他に「入力されるキーの確認」と「ユーザー設定」があります。

* 「入力されるキーの確認」：タップするエリアと入力されるキーの対応が確認できます。対応を見ながらahkファイルを作成・編集することもできます。
* 「ユーザー設定」：主にタップ判定に関する設定値を変更できます。中央エリアのサイズも設定できます。

# 判定エリア
判定エリアは以下の画像のように、中央・左上・右上・左下・右下の5つに別れています。
タッチパッドを上下左右に4等分して作られたエリアがあり、それらの上に円形の中央エリアが重ねて配置されています。
中央エリアは指2本でタップされた場合にのみ判定を行い、タップされた指が1本以下の場合は無視されます。
その場合、それぞれの指は他の4つのエリアのどこかに属していると見なされます。

<img width="400" height="225" alt="touchpad" src="https://github.com/user-attachments/assets/c794221c-e628-4e8e-9399-fc868877068f" />

* 指が2本とも同じエリアに属している場合：F13～F17（5通り）
  - 中央：F13
  - 左上：F14
  - 右上：F15
  - 左下：F16
  - 右下：F17
* 指がそれぞれ違うエリアに属している場合：F18～F23（6通り）
  - 左上・右上：F18
  - 左上・左下：F19
  - 左上・右下：F20
  - 右上・左下：F21
  - 右上・右下：F22
  - 左下・右下：F23

入力パターンとキーの対応は、前述の「入力されるキーの確認」からいつでも確認できます。
